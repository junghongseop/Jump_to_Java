<자바의 구조>

-클래스 블록-
public class 클래스명{

	-메서드 블록-
	[public|private|protected] [static] (리턴 자료형|void) 메서드명1(입력자료형 매개변수){
		명령문(statement);
	}

	-메서드 블록-
	[public|private|protected] [static] (리턴자료형|void) 메서드명2(입력자료형 매개변수){
		명령문(statement);
	}

}

소스 코드의 가장 바깥쪽 영역은 클래스 블록
클래스명은 원하는 이름으로 지을 수 있음, 소스파일의 이름(클래스명.java)과 동일하게 사용

클래스 블록은 매서드 블록들로을 포함

메서드 블록 부분에서 [public|private|protected]라고 되어 있는 부분의 의미는 public, private, protected또는 아무것도 오지 않을 수 있다는 의미
public, private, protected는 메서드의 접근제어자를 뜻함

[static]은 static 키워드가 올수도 있고 오지 않을 수도 있다는 의미
static이라는 키워드가 붙게 되면 static메서드가 됨

(리턴자료형|void)는 메서드가 실행된 후 리턴되는 값의 자료형을 의미
리턴값이 있을 경우에는 반드시 리턴 자료형을 표기해야 하며 만약 리턴값이 없는 경우라면 void로 표기해야 함
이 항목은 둘 다 생략할 수 없고 void 또는 리턴자료형이 반드시 있어야만 함

메서드명은 원하는 이름으로 지을수 있음
메서드 명 이후의 괄호() 안의 값들은 메서드의 입력 인자를 뜻함
입력 인자의 갯수는 제한이 없으며 입력 인자는 "입력자료형"+"매개변수형" 형태로 이루어 짐

-속성과 생성자-

public class Sample {

	-속성-
	private String message;

	-생성자-
	public Sample(String message) {
		this.message = message
	}
}
속성은 클래스의 상태를 나타내는 변수로 클래스 내부에서 선언
생성자는 클래스 인스턴스를 생성할 때 실행되는 메서드로 속성을 초기화하는 용도로 사용

-클래스 블록-
소스코드의 가장 바깥쪽은 클래스 블록임
클래스 블록은 중괄호로 둘러싸여야 함

{ - 블록의 시작
} - 블록의 끝

public은 자바의 접근제어자로 어디서든 이 클래스에 접근할 수 있음을 의미
class는 클래스 블록을 만드는 키워드임

-메서드 블록-
메서드 블록도 중괄호로 영역을 구분
클래스 블록 안에는 여러개의 메서드 블록이 있을 수 있음

static - 메서드에 static 키워드가 붙을 경우 아 메서드는 클래스 메서드가 되어 객체를 만들지 않아도 "클래스명.메서드명"형태로 호출이 가능
void - 메서드의 리턴타입 중 하나로 void는 리턴값이 없음을 의미
String[] args - 메서드의 매개 변수임. args변수는 String[] 배열 자료형임을 의미. args라는 이름은 인수를 의미하는 arguments의 약어로 관례적인 이름. args대신 다른 이름을 사용해도 상관 없음

-명령문-
컴퓨터에 무언가 일을 시키는 문장을 명령문이라고 함
명령문은 반드시 세미콜론을 붙여 문장의 끝을 표시해야 함
메서드 블록 안에는 여러개의 명령문이 있을 수 있음
